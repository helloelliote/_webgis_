# ------------- COMMIT START -------------
# <tag>: <제목> 적용될 경우, 이 커밋은... (필수)
# 예) feat: SCSS 를 CSS 변환없이 바로 임포트


# <본문> 이 변경 사항이 필요한 이유를 설명 (선택)
# 예) - CSS 의 불편을 해소하고, CSS 로 변환은 리액트가 자동으로 처리


# <참조> 관련 문서 또는 기타 리소스에 대한 링크 제공 (선택)
# 예) - [SASS](https://namu.wiki/w/Sass)


# -------------- COMMIT END --------------
# <tag> 는 다음 중 하나를 선택한다:
#  - feat     (새로운 기능 추가)
#  - fix      (버그 수정)
#  - refactor (코드 리팩토링)
#  - style    (서식, 누락된 세미콜론 등의 변경; 코드 변경 없음)
#  - chore    (빌드 부분 혹은 패키지 매니저 수정사항)
#  - doc      (문서 수정)
# ----------------------------------------
# 다음을 기억하여 작성한다:
#  - 제목 줄은 명령문 형식으로 작성한다
#  - 각 줄은 마침표로 끝내지 않는다
#  - <제목>, <본문>, <참조> 사이에는 빈 줄을 넣는다
#  - <본문> 은 "어떻게"가 아닌 "무엇"과 "왜"를 설명한다
#  - <본문> 에는 '-' 로 나열된 여러 줄을 사용할 수 있다
# ----------------------------------------
